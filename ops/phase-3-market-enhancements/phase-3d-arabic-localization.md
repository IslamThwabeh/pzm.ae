# Phase 3D - Arabic Localization Quality Pass

## Objective

Improve Arabic commercial readability for UAE users while preserving clarity, trust, and search discoverability.

## Standalone Mode

This phase can run independently on Arabic pages only.

## Inputs

1. Arabic target pages (5 pages)
2. Existing Arabic service style in ar/services/
3. Priority buyer concerns in Dubai (price, trust, stock, location)

## Deliverables

1. Improved Arabic tone for real local readability
2. Stronger conversion wording in CTA and pre-visit sections
3. Consistent terminology set for laptop intent pages

## Execution Checklist

1. Review each Arabic page for over-formal phrasing.
2. Rewrite critical CTA and conversion paragraphs into natural UAE-friendly Arabic.
3. Keep terminology consistent across all 5 Arabic pages.
4. Preserve factual meaning from English equivalents.
5. Run scripts/qa-page.ps1 on each changed page.

## Guardrails

1. Do not force slang everywhere; keep language professional and local-friendly.
2. Do not change technical meaning when simplifying language.
3. Keep page-level intent distinct.

## Definition Of Done

1. Arabic pages read naturally and consistently.
2. CTA clarity is stronger than baseline copy.
3. QA passes for all changed Arabic pages.

## Owner And Effort

1. Owner: Arabic content lead
2. Time: 0.5 to 1 day

## Rollback

If terminology appears too informal, revert specific phrases while keeping improved structure.
