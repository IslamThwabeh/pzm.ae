# Phase 3C - Intent Differentiation Across Pages

## Objective

Ensure each page has unique commercial intent and avoids template-like duplication across student, business, gaming, and MacBook pages.

## Standalone Mode

This phase can run independently and does not require policy changes.

## Inputs

1. Current EN + AR page copy for all 10 target pages
2. Existing internal links between service pages
3. Local query intent assumptions for Dubai buyers

## Deliverables

1. Distinct who-it-is-for section per page
2. Distinct budget-fit and use-case guidance per page
3. Strengthened cross-link map with intent-specific anchors

## Execution Checklist

1. Add a unique audience section to each page.
2. Add unique use-case bullets that do not repeat other pages.
3. Add at least 2 intent-specific internal links from each page.
4. Update intro paragraphs so each page targets one clear buying scenario.
5. Run scripts/qa-page.ps1 on each changed page.

## Internal Linking Rule

Use natural, scenario-based anchor text. Avoid identical anchor text reused everywhere.

## Definition Of Done

1. Manual side-by-side review confirms page uniqueness.
2. Every target page has at least 2 meaningful internal links.
3. QA passes for all changed pages.

## Owner And Effort

1. Owner: SEO + Content
2. Time: 1 to 1.5 days

## Rollback

If readability drops, keep unique sections but shorten length and simplify structure.
